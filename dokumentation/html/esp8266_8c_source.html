35  * +UART_DEF:<baudrate>,<databits>,<stopbits>,<parity>,<flow control>

37  *
38  * Set Command:
39  * AT+UART_DEF=<baudrate>,<databits>,<stopbits>,<parity>,<flow control>
40  * Response:
41  * OK
42  *
43  * Parameter:
44  * <baudrate>: UART baud rate
45  * <databits>: data bits
46  * 5: 5-bit data
47  * 6: 6-bit data
48  * 7: 7-bit data
49  * 8: 8-bit data
50  * <stopbits>: stop bits
51  * 1: 1-bit stop bit
52  * 2: 1.5-bit stop bit
53  * 3: 2-bit stop bit
54  * <parity>: parity bit
55  * 0: None
56  * 1: Odd
57  * 2: Even
58  * <flow control>: flow control
59  * 0: flow control is not enabled
60  * 1: enable RTS
61  * 2: enable CTS
62  * 3: enable both RTS and CTS
63  *
64  * Notes
65  * 1. The configuration changes will be saved in the user parameter area in the
66  * flash, and will still be valid when the chip is powered on again.
67  * 2. The use of flow control requires the support of hardware:
68  * MTCK is UART0 CTS
69  * MTDO is UART0 RTS
70  * 3. The range of baud rates supported: 110~115200*40.
71  *
72  * Example AT+UART_DEF=115200,8,1,0,3

792 /*
793  * File: softAP.h
794  * Author: krs
795  *
796  * Created on August 31, 2019, 8:03 PM
797  *
798  * SoftAP is an abbreviated term for "software enabled access point". This is
799  * software enabling a computer which hasn't been specifically made to be a
800  * router into a wireless access point. It is often used interchangeably with
801  * the term "virtual router".
802  *
803  * SoftAP is a common method of configuring Wi-Fi products without a display or
804  * input device, such as a Wi-Fi enabled appliance, home security camera,
805  * smart home product or any other IoT device. The process typically involves
806  * these steps:
807  *
808  1. * The headless device turns on a SoftAP Wi-Fi hotspot.
809  * 2. The user downloads a product-specific app on a smartphone, and the app
810  * then either uses the underlying Android or iOS operating system to connect
811  * to the SoftAP hotspot, or directs the user to connect the phone manually.
812  * 3. The app then asks the user for the user's private Wi-Fi network name
813  * (SSID) and passkey
814  * 4. The app sends the SSID and passkey to the headless device over the SoftAP
815  * connection.
816  * 5. The headless device then falls off the SoftAP network and joins the user's
817  * private Wi-Fi network.
818  *
819  * This process can work well, but there are two core problems. First, the
820  * process often requires the user to manually connect to the SoftAP network,
821  * which can be confusing for mainstream users. Second, if the user enters the
822  * passkey incorrectly, or if the phone gets disconnected from the SoftAP
823  * network for any reason, it is difficult for the app and device to smoothly
824  * recover, so the user is often left having to factory reset the device and
825  * start over. Third, different phones (hardware and OS versions) handle SoftAP
826  * differently, so the user experience varies dramatically -- especially with
827  * the wide variety of Android hardware and software.
828  */
